# v7.4.0 — 2026-05-04

## Added
- **Activity file** (50-0868): per-session build touch-file for external process monitoring
- **Consumer hooks** (50-0867): setup guidance and loop-guard for consumer hook configuration
- **Session identity** (50-0864): improved session identity tracking
- **Animation status** (15-763): animation status improvements
- **Custom name tags** (`name-tag/set`, `name-tag/get`): agents can set and retrieve a custom session header string that overrides the auto-default `<color> <name>` label
- **HTTP `/dequeue` endpoint** (GET and POST): non-MCP callers (curl, file-watchers, external processes) can consume the message queue over HTTP using the same session token
- **`profile/kick-debounce` action**: per-session override for the activity-file kick debounce window (30 s – 600 s)

## Fixed
- **`inflightDequeue` state machine**: `setDequeueActive(sid, true)` is now called only after the session-closed guard, preventing spurious kick suppression on closed sessions

## Changed
- **Default session header**: sessions no longer receive the `🤖` robot emoji prefix in their auto-computed name tag
- **Session close confirmation**: now includes the session name and SID for clarity
- **`nametag_emoji` profile field**: renamed to `name_tag`; profiles saved with the old field name are automatically migrated on load

## Removed
_Nothing removed._
